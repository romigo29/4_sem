Интернет:
1) стек протоколов TCP/IP
2) сетевые службы (службы Интернета) - программа-сервер, которая прослушивает сетевой порт в диапазоне от 0 до 1024 (хорошо известные порты)
3) Набор организаций, поддерживаюших работоспособность и развитие Internet:
- ISOC (Internet Society) – международная организация, занимающаяся развитием сети Internet
- IETF (Internet Engineering Task Force) - организация, инженерная группа, которая предназначена для проектирования
интернет, выпуска документации
- IAB (Internet Architecture Board) - совет по архитектуре Internet
- IANA (Internet Assigned Numbers Authority) - Администрация адресного пространства Internet.
- ICANN (Internet Corporation for Assigned Names and Numbers) – корпорация по управлению доменными именами и
 IP-адресами
- W3C (World Wide Web Consortium) – организация разрабатывающая и внедряющая web-стандарты (HTTP, HTML, URI/URL, CSS, DOM, XML, PNG, SVG,…). Возглавляет Тимоти Джон Бернерс-Ли.  
4) Документация (RFC, STP):
RFC (request for comments): формат документации в Интернете


Веб-приложение — клиент-серверное приложение, в котором клиент взаимодействует с сервером по протоколу HTTP.


DOM - внутренняя модель представления браузером html-страницы


Request:
- метод;
- URI;
- версия протокола (HTTP/1.1);
- заголовки (пары: имя/заголовок);
- параметры (пары: имя/заголовок);
- расширение


Response:
-версия протокола (HTTP/1.1);
-код состояния (1xx, 2xx, 3xx, 4xx, 5xx);
-пояснение к коду состояния;
-заголовки (пары: имя/заголовок);
-расширение


Response: Код состояния:
- 1xx: информационные сообщения;
- 2xx: успешный ответ;
- 3xx: переадресация;
- 4xx: ошибка клиента;
- 5xx: ошибка сервера

Методы Map:
GET (получение данных)
POST (добавление данных)
PUT (изменение данных)
DELETE (удаление данных)
